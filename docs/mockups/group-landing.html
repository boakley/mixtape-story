What changed since the last revision

  • Timeline removed. No more year column or dot/rail on song views — flat list, with year-on-the-rail dropped from the sort key. The personal mixtape page change is already live in SongRow.svelte.
  • Member cards compacted. Separators between cards gone; vertical padding dropped from 14px to 8px. Year range removed from the meta line (no longer relevant without the year-as-structure framing) — just song count.
  • Story truncation on song views. Songs we share + All songs now show the first 2 sentences of each story with an inline [more] link. The personal mixtape view at /{handle} still shows full stories.
  • All songs sort. Newest added first (was: oldest by year). Matches the writing-group's "what's new since I last looked" reading pattern.
  • Steward edit affordance. Small pencil at the end of the description, visible only to stewards. Clicking opens an inline edit (same pattern as the existing Edit mixtape link).

Story truncation rules

  • Default cap. First 2 sentences of the source Story.text. Sentence boundary = [.!?] followed by whitespace or end-of-string. If the first 2 sentences exceed ~280 characters, truncate at ~280 with ellipsis instead.
  • [more] behavior. Inline expand (no navigation away). Toggles to [less] when expanded. Re-uses the same expand affordance the personal mixtape page already has for compact-view stories.
  • When stories are shorter than the cap. No [more] shown; the full story renders inline.
  • Plain text, not HTML. Truncation operates on Story.text before markdown rendering — avoids splitting in the middle of a tag. The shortened text is then re-rendered.
  • Out of scope. Per-story-style customization; "Show all" bulk expand; remembering which stories you've already expanded across sessions.

Songs-we-share / All-songs rules (updated)

  • Definition. Songs we share = songs with ≥2 distinct member contributors. All songs = everything, including the shared subset.
  • Dedup key. ISRC when both rows have it; normalized title + artist as fallback.
  • Sort across songs. Newest added first (most recent Story.created_at across contributors, for each song). Replaces the previous "year-on-the-rail ascending" rule.
  • Story order under a song. Earliest Story.created_at first — preserves the thread feel.
  • memory_year still rendered inside stories as "This song reminds me of {year}." lead-in when present. Just no longer a structural sort key.

Steward controls — where they live

  • Description edit: small pencil at end of .group-desc, visible only when viewer.role === 'steward'. Inline edit pattern.
  • Other steward affordances (manage invite codes, member roster, delete group) live on a separate /g/{slug}/manage page accessed via a "Manage group" link in the hamburger menu when the viewer is a steward.
  • Non-stewards see nothing — no pencil, no Manage link. The role-conditional rendering happens server-side and is enforced by RLS underneath.

Still open

  • Tab persistence in localStorage like the expand/compact view on the mixtape page.
  • Whether the per-tab meta line ("9 mixtapes" / "47 unique songs") helps or just adds chrome. Could collapse to one consistent line ("12 members" only).
  • Should the "Songs we share" tab disappear when the group has zero shared songs, or stay with the empty state? Current lean: stay — the empty itself is a nudge.
  • Touch-target heights — tabs are currently ~33px tall; iOS-HIG suggests 44pt minimum. Worth a bump in the live build.
  • Does the steward pencil need a hover label ("Edit description") for discoverability, or is the inline placement enough? Hover-only on desktop is fine; on touch, a tap shows an inline editor without needing a label.
